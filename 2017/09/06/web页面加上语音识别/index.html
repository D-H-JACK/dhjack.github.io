web页面加上语音识别

发表于 2017-09-06

起因

开始做的电影推荐系统，只有两个按键反馈：喜欢键和不喜欢键。但是实际的使用中，用户应该是有多种述求的。比如不喜欢某个演员、想看科幻片等等。当然，主流的网站，比如豆瓣，做了详尽的分类导航系统，用户可以自由的选择。好吧，比较懒，不太像做这么复杂。联想到最近很热门的智能音箱，为什么不给页面加上一个语义控制模块了。这样用户可以随心所欲的“输入”，充分的个性化。所以，第一步就是让页面具有语音识别的能力。

过程

第一个想到的就是科大讯飞，尝试了下sdk，蛮容易上手的。不过好像只有C的接口？而目前的项目是在Flask上面开发的，如果要用，还得转换下。然后看到百度居然有Rest api。对这种使用过程很感兴趣，感觉很灵活。所以开始的第一个想法是，页面直接录制到音频数据，然后通过百度Rest api获取结果。然后发现不行。

  • 暴露账号信息
  • 音频码率问题，页面直接录制的音频格式是44100HZ的wav格式，而百度api有音频格式要求：
    • 原始语音的录音格式目前只支持评测 8k/16k 采样率 16bit 位深的单声道语音
    • 压缩格式支持：pcm（不压缩）、wav、amr
    • 系统支持语言种类：中文（zh）、粤语（ct）、英文（en）。
  • 跨域名访问

那么就通过ajax把音频数据发送给服务器，服务器先转换码率，然后再调用百度api识别，并返回给页面。

后续
